## 1. LIFF（LINE内ブラウザ）UX設計指針

### 1.1 LIFF概要と制約

LineMagの顧客向け画面はすべてLIFF（LINE Front-end Framework）上で動作する。LIFFアプリはLINEアプリ内のWebViewで表示されるため、通常のブラウザとは異なる制約・特性を持つ。

| 項目 | 内容 |
|------|------|
| 表示モード | `Full`（全画面表示）を標準採用 |
| 認証 | LIFF SDKの `liff.getProfile()` でLINEユーザー情報を自動取得 |
| ビューポート | LINE内WebView（iOS/Android）。セーフエリアを考慮 |
| 戻る操作 | LINEアプリのネイティブ「戻る」ボタンを利用可能 |
| 外部遷移 | `liff.openWindow()` で外部ブラウザに遷移（Google Meet等） |

### 1.2 UX設計原則

#### 原則1: ログインレス体験
- LIFF初期化時に `liff.init()` → `liff.isLoggedIn()` で自動認証
- ユーザーにID/パスワード入力を求めない
- LINEプロフィール（表示名・アイコン）をそのまま利用

#### 原則2: 最小タップ数の追求
- 主要操作は3タップ以内で完了を目標とする
- 予約完了まで: カレンダー日付タップ → スロットタップ → 確認ボタン → 完了（4タップ）
- クーポン利用: クーポンタップ → 利用ボタン → 完了（3タップ）

#### 原則3: LINEネイティブとの一体感
- LINE公式カラー（`#06C755`）をアクセントカラーとして使用
- フォントサイズ・行間はLINEアプリのメッセージ画面と近い感覚に
- ボタン形状は角丸（`rounded-xl`）を基本とする

#### 原則4: 即時フィードバック
- タップ後は100ms以内に視覚的フィードバック（`active:scale-95` 等）
- API通信中はスケルトンスクリーンまたはスピナーを表示
- 完了時はチェックマークアニメーションで達成感を演出

### 1.3 LIFF初期化フロー

```
LINEトーク内リンクタップ
  → LIFFアプリ起動
  → liff.init({ liffId })
  → liff.isLoggedIn() === true ?
    → YES: liff.getProfile() でユーザー情報取得 → メイン画面表示
    → NO:  liff.login() → 認証後リダイレクト → メイン画面表示
```

### 1.4 画面遷移の基本パターン

```
リッチメニュー / メッセージ内リンク
  ├── /liff/reservation  → 予約画面
  ├── /liff/coupons      → クーポン一覧
  ├── /liff/mypage        → マイページ
  └── /liff/articles      → ブログ記事一覧
```

## 2. 予約フロー画面設計

### 2.1 概要

相談窓口の予約機能。ユーザーが日付・時間枠を選択し、Google Meetでのオンライン相談を予約する。

| 項目 | 仕様 |
|------|------|
| スロット種別 | 30分 / 1時間 |
| 会議方法 | Google Meet（自動生成リンク） |
| 予約確認 | LINE通知 + 予約完了画面 |
| キャンセル | マイページ予約履歴から（24時間前まで） |

| LINE通知 | Messaging APIで予約確認メッセージを自動送信 |
| 「トークに戻る」 | `liff.closeWindow()` でLIFFを閉じてトーク画面に戻る |

### 2.3 予約関連の状態管理

```
状態遷移:
  IDLE → DATE_SELECTED → SLOT_SELECTED → CONFIRMING → COMPLETED
                                              ↓
                                           FAILED (エラー時)
```

| 状態 | 画面 | 保持データ |
|------|------|----------|
| IDLE | Step 1 | 相談タイプ |
| DATE_SELECTED | Step 1 → Step 2 遷移 | 相談タイプ + 選択日 |
| SLOT_SELECTED | Step 2 → Step 3 遷移 | 相談タイプ + 選択日 + スロット |
| CONFIRMING | Step 3 API呼出中 | 全データ + 連絡事項 |
| COMPLETED | Step 4 | 予約ID + Google Meet URL |
| FAILED | Step 3 エラー表示 | エラーメッセージ |

## 6. ユーザーフロー図

### 6.1 全体フロー概要

```
LINE公式アカウント
  │
  ├── リッチメニュー
  │     ├── [予約する]   ──→ 予約フロー (6.2)
  │     ├── [クーポン]   ──→ クーポンフロー (6.3)
  │     ├── [マイページ] ──→ マイページフロー (6.4)
  │     └── [記事を読む] ──→ 記事フロー (6.5)
  │
  └── プッシュメッセージ
        ├── 新着記事通知  ──→ 記事詳細画面
        ├── クーポン配布  ──→ クーポン詳細画面
        └── 予約リマインド ──→ 予約詳細画面
```

### 6.2 予約フロー

```
[リッチメニュー: 予約する]
  │
  ▼
LIFF起動 → liff.init()
  │
  ▼
相談タイプ選択（30分 / 1時間）
  │
  ▼
カレンダーで日付選択
  │
  ├──(予約可能日あり)──→ スロット一覧表示
  │                        │
  │                        ├──(スロット選択)──→ 確認画面
  │                        │                      │
  │                        │                      ├──(確定)──→ API送信
  │                        │                      │              │
  │                        │                      │              ├──(成功)──→ 完了画面
  │                        │                      │              │              │
  │                        │                      │              │              ├─→ LINE通知送信
  │                        │                      │              │              ├─→ Google Meet URL表示
  │                        │                      │              │              └─→ マイページ / トーク戻る
  │                        │                      │              │
  │                        │                      │              └──(失敗)──→ エラー表示
  │                        │                      │                              │
  │                        │                      │                              └─→ スロット再選択へ
  │                        │                      │
  │                        │                      └──(戻る)──→ スロット一覧へ
  │                        │
  │                        └──(戻る)──→ カレンダーへ
  │
  └──(予約可能日なし)──→ 「予約可能な日がありません」表示
```

### 6.3 クーポンフロー

```
[リッチメニュー: クーポン] or [プッシュメッセージ: クーポン配布]
  │
  ▼
LIFF起動 → liff.init()
  │
  ▼
クーポン一覧画面
  │
  ├──[利用可能タブ]
  │    │
  │    └──(クーポンタップ)──→ クーポン詳細画面
  │                             │
  │                             ├──[クーポンコードコピー]──→ コピー完了トースト
  │                             │
  │                             └──[このクーポンを使う]
  │                                  │
  │                                  ▼
  │                                確認ダイアログ
  │                                  │
  │                                  ├──(利用する)──→ API送信
  │                                  │                  │
  │                                  │                  ├──(成功)──→ 利用完了表示
  │                                  │                  │              └─→ LINE通知送信
  │                                  │                  │
  │                                  │                  └──(失敗)──→ エラートースト
  │                                  │
  │                                  └──(キャンセル)──→ 詳細画面に戻る
  │
  └──[利用済みタブ]
       │
       └──(クーポンタップ)──→ 利用済み詳細画面（利用日時表示）
```

### 6.4 マイページフロー

```
[リッチメニュー: マイページ]
  │
  ▼
LIFF起動 → liff.init()
  │
  ▼
マイページトップ
  │
  ├──[プロフィール編集]──→ プロフィール編集画面

  │
  ├──[予約履歴]──→ 予約履歴画面

  │                  ├──[今後の予約タブ]

  │                  │    ├──[Google Meet]──→ 外部ブラウザでMeet起動
  │                  │    │
  │                  │    └──[取消]──→ 確認ダイアログ → キャンセルAPI → LINE通知
  │                  │
  │                  └──[過去の予約タブ]──→ 履歴一覧表示
  │
  ├──[クーポン一覧]──→ クーポン一覧画面 (6.3と同一)
  │
  └──[お気に入り記事]──→ ブックマーク済み記事一覧
                           │
                           └──(記事タップ)──→ 記事詳細画面
```

### 6.5 記事閲覧フロー

```
[リッチメニュー: 記事を読む] or [プッシュメッセージ: 新着記事通知]
  │
  ▼

  │
  ▼
記事一覧画面 /liff/articles
  │
  ├──(カテゴリフィルタ)──→ フィルタ済み一覧
  │
  └──(記事タップ)──→ 記事詳細画面 /liff/articles/[id]
                        │
                        ├──[お気に入り登録]──→ ブックマーク保存
                        │
                        ├──[LINEで共有]──→ liff.shareTargetPicker()
                        │
                        └──[関連記事]──→ 別の記事詳細画面
```

### 6.6 エラーハンドリングフロー

```
全画面共通エラーハンドリング:

LIFF初期化エラー
  └──→ 「LINEアプリから開いてください」画面
       └──[LINEで開く]──→ LINE URLスキームで誘導

ネットワークエラー
  └──→ インライン通知バー「接続エラー。再試行してください」
       └──[再試行]──→ API再呼出

認証エラー (トークン期限切れ)
  └──→ liff.login() で再認証

API 4xx エラー
  └──→ 操作に応じたエラーメッセージをトースト表示

API 5xx エラー
  └──→ 「只今メンテナンス中です」画面
       └──[トークに戻る]──→ liff.closeWindow()
```
